## 문제 설명
2025년 10월 14일 화요일 오전 9시(KST)에 스케줄된 워크플로우가 자동으로 실행되지 않았습니다.

## 현재 설정
- 스케줄: `0 0 * * 1-5` (UTC 00:00 = KST 09:00, 평일만)
- 워크플로우 파일: `.github/workflows/daily-menu.yml`

## 관찰된 현상
- 최근 실행 기록이 모두 `workflow_dispatch` (수동 실행)으로만 표시됨
- 스케줄에 의한 자동 실행 기록이 없음
- 현재 시각: 2025-10-14 00:30 UTC (09:30 KST)

## 가능한 원인
1. **GitHub Actions 스케줄 지연/누락**: GitHub Actions의 cron은 정확한 시간을 보장하지 않으며, 높은 부하 시간대에는 지연되거나 건너뛸 수 있습니다
2. **리포지토리 비활성화**: 60일 이상 활동이 없는 공개 리포지토리의 경우 스케줄된 워크플로우가 자동으로 비활성화될 수 있습니다
3. **워크플로우 권한 문제**: Actions 권한 설정에 문제가 있을 가능성

## 해결 방안
- [x] GitHub Actions 탭에서 워크플로우가 활성화되어 있는지 확인
- [x] 리포지토리 설정에서 Actions 권한 확인
- [x] 워크플로우가 기본 브랜치(main)에 병합되어 있는지 확인 ⭐ **핵심 해결책**

## 근본 원인 분석

**주요 원인**: GitHub Actions의 스케줄된 워크플로우는 **오직 기본 브랜치(main/master)에서만 실행**됩니다.

1. 워크플로우 파일이 PR 브랜치나 다른 브랜치에만 있는 경우 스케줄 실행이 작동하지 않습니다
2. 워크플로우를 기본 브랜치에 병합한 후에야 스케줄 실행이 시작됩니다
3. 첫 스케줄 실행은 병합 후 다음 예정 시간(다음날 오전 9시)에 발생합니다

**부차적 원인**:
- 워크플로우가 오전 9:31 KST에 생성되어 당일 오전 9:00 KST 실행 시간을 놓침
- 리포지토리가 60일 이상 비활성 상태인 경우 스케줄이 자동으로 비활성화됨
- GitHub Actions의 cron 스케줄은 높은 부하 시간대에 지연되거나 건너뛸 수 있음
